四路独立按键驱动模块详解

BSP/bsp_key.c + BSP/bsp_key.h  |  STM32F103 智能浇花系统

1. 模块概述

本模块驱动 4 路独立物理按键，采用状态机消抖扫描算法，提供单次/连续按压两种模式。每个按键对应一个独立 GPIO 引脚，通过轮询方式检测电平变化。

按键引脚GPIO 端口功能
KEY1PB0GPIOB模式切换（自动/手动/阈值设）
KEY2PA7GPIOA阈值 +5%
KEY3PA3GPIOA阈值 -5%
KEY4PA5GPIOA从 Flash 载入保存的阈值
电路说明： 按键按下时引脚为高电平（GPIO_Mode_IPD 内置下拉电阻），松开时为低电平。无需外部上拉/下拉电阻。

2. 硬件初始化 — Key_Init

将各按键引脚配置为下拉输入模式（GPIO_Mode_IPD），使能 GPIOA 和 GPIOB 时钟。

void Key_Init(void)
{
    GPIO_InitTypeDef GPIO_InitStructure;

    RCC_APB2PeriphClockCmd(RCC_APB2Periph_GPIOA | RCC_APB2Periph_GPIOB, ENABLE);

    // PA7(KEY2) + PA3(KEY3) + PA5(KEY4)：下拉输入
    GPIO_InitStructure.GPIO_Pin  = KEY2_PORT_PIN | KEY3_PORT_PIN | KEY4_PORT_PIN;
    GPIO_InitStructure.GPIO_Mode = GPIO_Mode_IPD;
    GPIO_Init(GPIOA, &GPIO_InitStructure);

    // PB0(KEY1)：下拉输入
    GPIO_InitStructure.GPIO_Pin  = KEY1_PORT_PIN;
    GPIO_Init(GPIOB, &GPIO_InitStructure);
}
为什么用 IPD？ 下拉输入确保按键松开时引脚为确定的低电平，按下时外部高电平可以可靠被检测到。相比外部下拉电阻，内置下拉节省 PCB 空间和 BOM 成本。

3. 状态机消抖扫描 — Key_Scan

核心扫描函数，基于释放确认状态机实现按键消抖，返回 1~4 表示对应按键按下，0 表示无按键。

// mode: 0=不支持连按, 1=支持连按
// 返回值: 0=无按键, 1=KEY1, 2=KEY2, 3=KEY3, 4=KEY4
// 优先级: KEY1 > KEY2 > KEY3 > KEY4
uint8_t Key_Scan(uint8_t mode)
{
    static uint8_t key_up = 1;       // 1=按键已释放, 0=按键尚未释放

    if (mode) key_up = 1;            // 连按模式：强制标记可再次检测

    if (key_up && (KEY1==1 || KEY2==1 || KEY3==1 || KEY4==1))
    {
        DelayXms(10);                  // ★ 10ms 延时消抖
        key_up = 0;                    // 标记按下，防止重复触发

        if (KEY1 == 1) return 1;
        else if (KEY2 == 1) return 2;
        else if (KEY3 == 1) return 3;
        else if (KEY4 == 1) return 4;
    }
    else if (KEY1==0 && KEY2==0 && KEY3==0 && KEY4==0)
    {
        key_up = 1;                    // 所有按键释放，恢复检测
    }
    return 0;
}
状态机原理：
状态条件动作
待检测 (key_up=1)检测到任意按键 →延时 10ms 消抖，确认后返回键值，进入"已按下"状态
已按下 (key_up=0)有按键保持按下不返回任何键值（阻止重复触发）
恢复待检测 (key_up=1)全部按键释放重置状态，允许下次按下被识别
连按模式 (mode=1)：检测到按键后立即将 key_up 强制设为 1，实现长按时连续返回键值
优先级说明： 当多个按键同时按下时，函数按 KEY1 → KEY2 → KEY3 → KEY4 的顺序响应，仅返回优先级最高的键值。

4. 按键读取宏定义

通过 GPIO_ReadInputDataBit 读取各引脚电平，封装为类函数宏，方便在扫描函数中直接使用。

#define KEY1  GPIO_ReadInputDataBit(GPIOB, GPIO_Pin_0)
#define KEY2  GPIO_ReadInputDataBit(GPIOA, GPIO_Pin_7)
#define KEY3  GPIO_ReadInputDataBit(GPIOA, GPIO_Pin_3)
#define KEY4  GPIO_ReadInputDataBit(GPIOA, GPIO_Pin_5)
宏 vs 函数： 使用宏定义省去了函数调用开销，适合按键扫描这种高频调用的场景。实际编译后，GPIO_ReadInputDataBit 会被内联为位带操作或寄存器读取，效率极高。

6. 按键功能汇总表

按键GPIO引脚宏功能场景
KEY1GPIOBPB0KEY1 模式切换自动模式 ↔ 手动模式 ↔ 阈值设置
KEY2GPIOAPA7KEY2 阈值 +5%阈值设置模式下增加湿度阈值
KEY3GPIOAPA3KEY3 阈值 -5%阈值设置模式下减少湿度阈值
KEY4GPIOAPA5KEY4 Flash 载入从 Flash 读取保存的阈值并应用
按键复用逻辑： KEY2/KEY3/KEY4 在不同模式下具有不同含义。阈值设置模式下，KEY2 和 KEY3 修改阈值；在自动/手动模式下可用于切换显示页面或触发手动浇水。
