主循环调度与控制逻辑模块详解

文件：Main/main.c（316 行） |  系统入口与核心调度

1. 模块概述

main.c 是整个智能灌溉系统的入口文件和核心控制逻辑。它负责：

  • 初始化所有外设（GPIO、USART、ADC、OLED、按键、水泵、报警器等）
  • 连接 WiFi 并登录 OneNet 云平台
  • 订阅下行主题以接收远程控制命令
  • 进入无限循环，通过轮询调度执行 6 个周期性任务
调度策略：采用计数轮询方式，不使用 RTOS。每个主循环周期约 10ms（基于 Delay_ms(10)），通过 TimeCount 和 PingCount 计数器实现不同频率的任务调度。
阶段动作
初始化Delay, GPIO, USART1/USART2, ADC, OLED, Key, WaterSensor, Alarm, Motor
Flash 加载从 0x0800FC00 读取上次保存的阈值
WiFi 连接ESP8266_Init + ESP8266_Connect，失败则重试
OneNet 登录OneNet_DevLink + OneNet_Subscribe
主循环6 个任务按固定频率轮询执行

3. ADC 读取与转换

Read_ADC — 读取 DMA 缓冲区

uint16_t Read_ADC(void)
{
  uint32_t sum = 0;
  for (uint8_t i = 0; i < 10; i++) {
    sum += ADC_Value[i];      // 读取 DMA 循环缓冲区，取 10 次均值
  }
  return sum / 10;
}

ADC_To_Humidity — 电压转湿度百分比

uint16_t ADC_To_Humidity(uint16_t adcValue)
{
  // 土壤湿度传感器电压范围：0V(干燥) ~ 3.3V(湿润)
  // STM32 ADC 12位：0 ~ 4095 对应 0 ~ 3.3V
  // 线性映射到 0% ~ 100%，并做限幅

  uint16_t humidity = (adcValue * 100) / 4095;

  if (humidity > 100) humidity = 100;  // 上限钳位
  if (humidity < 0)   humidity = 0;    // 下限钳位

  return humidity;
}
线性映射关系：湿度(%) = (ADC值 / 4095) × 100。实际使用时可根据传感器特性进行标定校准，替换为查表或多项式拟合。

4. Flash 阈值存储

阈值数据保存在 STM32 内部 Flash 的最后一个页面（0x0800FC00），写入前使用 magic number 0xA5A5 校验数据有效性：

#define FLASH_THRESHOLD_ADDR  0x0800FC00
#define MAGIC_NUMBER          0xA5A5
/* 保存阈值到 Flash */
void Threshold_Save(uint16_t threshold)
{
  FLASH_Unlock();
  FLASH_ErasePage(FLASH_THRESHOLD_ADDR);       // 擦除页面
  FLASH_ProgramHalfWord(FLASH_THRESHOLD_ADDR,       MAGIC_NUMBER);  // 写入 Magic
  FLASH_ProgramHalfWord(FLASH_THRESHOLD_ADDR + 2,   threshold);     // 写入阈值
  FLASH_Lock();
}

/* 从 Flash 加载阈值 */
uint16_t Threshold_Load(void)
{
  uint16_t magic = *(volatile uint16_t *)FLASH_THRESHOLD_ADDR;
  if (magic == MAGIC_NUMBER) {
    return *(volatile uint16_t *)(FLASH_THRESHOLD_ADDR + 2);
  }
  return 50;  // 默认阈值 50%
}
Flash 写入注意：Flash 写入前必须执行擦除操作。每次擦除以页为单位（STM32F103 每页 1KB 或 2KB）。擦除操作耗时约 20ms，在此期间系统应避免响应中断。

5. 水泵滞回控制 — Motor_Control_Check

水泵控制采用滞回比较策略，防止湿度在阈值附近频繁启停：

void Motor_Control_Check(uint16_t humidity)
{
  if (humidity < Humidity_Threshold) {
    // 湿度低于阈值 → 开启水泵
    GPIO_ResetBits(MOTOR_PORT, MOTOR_PIN);    // 低电平驱动继电器
    motor_running = 1;

    // 只有正在浇灌时才将报警标志置 1，并发布报警状态
    if (!Alarm_flag) {
      Alarm_flag = 1;
    }

  } else {
    // 湿度 >= 阈值：启动延时关闭计数器
    if (motor_running) {
      motor_timer++;
      if (motor_timer >= 400) {        // 约 4 秒
        GPIO_SetBits(MOTOR_PORT, MOTOR_PIN);    // 关闭水泵
        motor_running = 0;
        motor_timer = 0;
        Alarm_flag = 0;                  // 关闭报警
      }
    }
  }
}
滞回效果：当湿度低于阈值时立即开启水泵；当湿度恢复到阈值以上后，水泵继续运行约 4 秒（400 个主循环）才关闭，确保土壤充分吸水，避免在阈值边界处频繁切换。

7. 按键处理 — Handle_Keys

系统支持 4 个物理按键，功能分配如下：

按键正常模式设置模式
KEY1进入设置模式退出并保存阈值 + 发布到 OneNet
KEY2—阈值 +5%
KEY3—阈值 -5%
KEY4从 Flash 加载阈值—
 
void Handle_Keys(void)
{
  // KEY1: 设置模式开关
  if (KEY1_IsPressed()) {
    Delay_ms(20);                    // 消抖
    if (KEY1_IsPressed()) {
      if (!setting_mode) {
        setting_mode = 1;
      } else {
        // 退出设置模式：保存到 Flash 并发布到云平台
        Threshold_Save(Humidity_Threshold);
        setting_mode = 0;

        // 发布更新后的阈值到 OneNet
        sprintf(PUBLIS_BUF,
          "{\"id\":\"123\",\"params\":{\"Threshold\":{\"value\":%d}}}",
          Humidity_Threshold);
        OneNet_Publish(PUBLIS_BUF);
      }
    }
  }

  // KEY2: 阈值 +5%（仅设置模式有效）
  if (KEY2_IsPressed() && setting_mode) {
    Delay_ms(20);
    if (KEY2_IsPressed()) {
      if (Humidity_Threshold <= 95)
        Humidity_Threshold += 5;
    }
  }

  // KEY3: 阈值 -5%（仅设置模式有效）
  if (KEY3_IsPressed() && setting_mode) {
    Delay_ms(20);
    if (KEY3_IsPressed()) {
      if (Humidity_Threshold >= 5)
        Humidity_Threshold -= 5;
    }
  }

  // KEY4: 从 Flash 加载阈值
  if (KEY4_IsPressed()) {
    Delay_ms(20);
    if (KEY4_IsPressed()) {
      Humidity_Threshold = Threshold_Load();
    }
  }
}
消抖处理：每个按键检测都包含 20ms 延时消抖。由于主循环本身约 10ms，首次检测和消抖后二次确认可有效避免按键抖动导致的误触发。

/* ============== 主函数入口 ============== */
int main(void)
{
  /* 第一阶段：外设初始化 */
  Delay_Init();                  // 系统延时初始化
  GPIO_Remap();                  // GPIO 重映射（USART/ADC 引脚）
  USART1_Init(115200);           // 调试串口（PC 通信）
  USART2_Init(115200);           // ESP8266 通信串口
  ADC_Init();                     // ADC + DMA 初始化（土壤湿度传感器）
  OLED_Init();                    // 0.96寸 OLED 初始化
  KEY_Init();                     // 按键初始化
  WaterSensor_Init();             // 水位传感器初始化
  Alarm_Init();                   // 蜂鸣器报警初始化
  Motor_Init();                   // 水泵电机控制初始化

  /* 第二阶段：加载阈值 */
  Humidity_Threshold = Threshold_Load();
  OLED_ShowMsg("Threshold Loaded");
  Delay_ms(1000);

  /* 第三阶段：连接 WiFi */
  OLED_ShowMsg("WiFi Connecting...");
  while (ESP8266_Init() != 0) {    // 初始化 ESP8266
    Delay_ms(500);
  }
  while (ESP8266_Connect() != 0) { // 连接 WiFi（SSID/PWD 在 esp8266.h 中）
    Delay_ms(500);
  }
  OLED_ShowMsg("WiFi Connected!");
  Delay_ms(1000);

  /* 第四阶段：登录 OneNet 并订阅 */
  OLED_ShowMsg("OneNet Linking...");
  while (OneNet_DevLink() != 0) {   // MQTT CONNECT（包含重试）
    Delay_ms(500);
  }
  OLED_ShowMsg("OneNet Link OK!");
  Delay_ms(500);

  OneNet_Subscribe();              // 订阅下行属性设置主题

  /* 第五阶段：进入主循环 */
  OLED_Clear();
  while (1)
  {
    /* 任务 1 — 接收 ESP8266 数据并解析 */
    uint8_t *recvBuf = ESP8266_GetIPD(10);
    if (recvBuf) {
      OneNet_RevPro(recvBuf);        // 核心解析函数
      free(recvBuf);                  // 释放由 GetIPD 分配的缓冲区
    }

    /* 任务 2 — 按键扫描 */
    Handle_Keys();

    /* 任务 3 — 水位传感器报警检测 */
    if (WaterSensor_IsEmpty()) {
      Alarm_ON();                     // 水箱缺水 → 蜂鸣器报警
    } else {
      Alarm_OFF();
    }

    /* 任务 4 — 水泵控制（滞回算法，每循环执行） */
    uint16_t adcVal = Read_ADC();
    uint16_t humidity = ADC_To_Humidity(adcVal);
    Motor_Control_Check(humidity);

    /* 任务 5 — 数据上传（每 100 循环 = ~1 秒） */
    TimeCount++;
    if (TimeCount >= 100) {
      TimeCount = 0;

      // 构建 JSON 数据
      sprintf(PUBLIS_BUF,
        "{\"id\":\"123\",\"params\":{"
        "\"Humidity\":{\"value\":%d},"
        "\"Threshold\":{\"value\":%d},"
        "\"alarm\":{\"value\":%s}}"
        "}",
        humidity, Humidity_Threshold,
        Alarm_flag ? "true" : "false");

      OneNet_Publish(PUBLIS_BUF);   // 上传到 OneNet

      // OLED 显示（根据设置模式选择不同界面）
      if (setting_mode) {
        OLED_Display_Setting(Humidity_Threshold);
      } else {
        OLED_Display_Normal(humidity, Humidity_Threshold,
          Alarm_flag, motor_running);
      }
    }

    /* 任务 6 — MQTT 心跳（每 3000 循环 = ~30 秒） */
    PingCount++;
    if (PingCount >= 3000) {
      PingCount = 0;
      OneNet_SendPing();
    }

    Delay_ms(10);                    // 每个主循环周期约 10ms
  }
}

9. 时序分析

主循环基于 Delay_ms(10)，每个循环周期约 10ms。任务调度通过计数器实现多级时间分片：

任务调度周期计数器计数目标实际间隔
ESP8266 数据接收每循环—1~10ms
按键扫描每循环—1~10ms
水位报警检测每循环—1~10ms
水泵控制每循环—1~10ms
数据上传 + OLED 刷新每 100 循环TimeCount100~1 秒
MQTT 心跳每 3000 循环PingCount3000~30 秒
水泵关闭延时每循环motor_timer400~4 秒
时间精度：上述时间基于 Delay_ms(10) 在理想情况下的估计。实际周期受限于 ESP8266 数据接收等待时间、Flash 写入耗时等因素。若通信占用时间较长，实际上传间隔会略大于 1 秒。
优化建议：如需更精确的时序控制，可将 Delay_ms(10) 替换为 SysTick 定时器中断，使用标志位驱动任务调度。当前轮询方式在代码简单性和时序精确性之间取得了平衡。
